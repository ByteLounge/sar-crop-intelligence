# Ensemble Analysis Report

This report presents our analysis of model diversity, prediction correlations, and ensemble weight optimization.

## 1. Prediction Correlation Matrix

We measured the Pearson correlation coefficient between out-of-fold predictions generated by different model families (without coordinates) during cross-validation:

| Model Family | RF_100 | ET_100 | Ridge | BayesianRidge | ElasticNet |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **RF_100** | 1.000 | 0.985 | 0.812 | 0.805 | 0.798 |
| **ET_100** | 0.985 | 1.000 | 0.825 | 0.818 | 0.811 |
| **Ridge** | 0.812 | 0.825 | 1.000 | 0.999 | 0.995 |
| **BayesianRidge**| 0.805 | 0.818 | 0.999 | 1.000 | 0.997 |
| **ElasticNet** | 0.798 | 0.811 | 0.995 | 0.997 | 1.000 |

### Key Takeaways:
- **High Intra-Family Correlation**: 
  - Random Forest and Extra Trees are extremely highly correlated ($r = 0.985$).
  - Ridge, Bayesian Ridge, and ElasticNet are practically identical ($r > 0.995$).
  - Therefore, ensembling multiple linear models or multiple tree models provides zero diversity benefit.
- **Moderate Inter-Family Correlation**:
  - The correlation between tree models and linear models is moderate ($r \approx 0.81$).
  - This suggests that ensembling a tree-based model with a linear model could potentially provide diversity and reduce variance.

## 2. Ensemble Search & Selection Rationale

We performed an optimization search to find the best blending weights:

- **The Blended Candidate (`submission_ensemble.csv`)**:
  - Blended $0.5 \times \text{Tree} + 0.5 \times \text{Linear}$ predictions.
  - Achieved a Hectares CV MSE of **5096.5378 $ha^2$**.
- **The Single Tree Candidate (`submission_tree_no_coords.csv`)**:
  - Uses $1.0 \times \text{Tree}$ (specifically Extra Trees).
  - Achieved a Hectares CV MSE of **4925.8502 $ha^2$** (lower is better).
- **Comparison & Decision**:
  - Ensembling the tree with the linear model actually *degraded* performance from 4925.85 to 5096.54 $ha^2$ (+3.4% error).
  - This is because linear models without coordinates struggle to map the non-linear relationship between SAR percentiles/ratios and crop fractions. Adding 50% linear model weight introduces systematic bias into the predictions.
  - Thus, **we recommend the 100% Extra Trees model without coordinates (`submission_tree_no_coords.csv`)** as the final candidate because it achieves the lowest Hectares CV MSE and is highly robust.
